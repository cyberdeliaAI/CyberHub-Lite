# Contributing to CyberHub Lite

Thanks for your interest — contributions are genuinely welcome. 🙏

## How to contribute

1. **Open an issue first** for anything non-trivial, so we can agree on the approach
   before you invest time.
2. Fork, branch, and keep PRs focused — one logical change per PR.
3. **Match the surrounding code** — same style, naming, and comment density. No new
   dependencies without discussing it first (Lite intentionally stays lightweight).
4. Test your change against a real image folder before submitting.

Good areas to contribute: metadata-format parsers, gallery/search UX, viewer/compare
improvements, integrations (e.g. sending generation data to Forge / ComfyUI).
